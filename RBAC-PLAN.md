# 🔐 ПЛАН РАЗДЕЛЕНИЯ РОЛЕЙ (RBAC)

## 📋 ТЕХНИЧЕСКОЕ ЗАДАНИЕ

### Цель:
Разделить интерфейсы и права доступа для трех ролей: BROKER, DEVELOPER, ADMIN

---

## 👥 РОЛИ И ПРАВА ДОСТУПА:

### 1. BROKER (Брокер)
**Задача:** Работа с клиентами и продажа квартир

#### Доступ к модулям:
- ✅ **CRM (Клиенты)** - CRUD только СВОИХ клиентов
- 👁️ **Новостройки** - только просмотр (без редактирования)
- 👁️ **Шахматка** - только просмотр квартир
- ✅ **Бронирование** - создание броней на любые квартиры
- ✅ **Брони** - управление СВОИМИ бронями
- ✅ **Продажа** - оформление продажи СВОИХ броней
- ✅ **Калькулятор** - расчет ипотеки
- 📊 **Аналитика** - только своя статистика

#### НЕ может:
- ❌ Создавать/редактировать проекты
- ❌ Создавать/редактировать квартиры
- ❌ Видеть клиентов других брокеров
- ❌ Управлять бронями других брокеров
- ❌ Создавать пользователей

---

### 2. DEVELOPER (Девелопер)
**Задача:** Создание проектов и квартир, контроль продаж

#### Доступ к модулям:
- ✅ **Новостройки** - CRUD СВОИХ проектов
- ✅ **Квартиры** - CRUD квартир в СВОИХ проектах
- 👁️ **Брони** - просмотр всех броней на СВОИ квартиры
- 👁️ **Видит какой брокер** какую квартиру забронировал
- ✅ **Продажа** - может оформить продажу на свои квартиры
- 📊 **Аналитика** - статистика по своим проектам

#### НЕ может:
- ❌ Видеть/создавать клиентов (нет доступа к CRM)
- ❌ Редактировать проекты других девелоперов
- ❌ Редактировать квартиры других девелоперов
- ❌ Создавать пользователей

---

### 3. ADMIN (Администратор)
**Задача:** Полное управление системой

#### Доступ к модулям:
- ✅ **Все модули** - полный доступ
- ✅ **Пользователи** - создание/редактирование брокеров и девелоперов
- ✅ **Все данные** - видит все клиенты, проекты, брони
- ✅ **Редактирование** - может редактировать любые данные
- ✅ **Аналитика** - полная статистика системы

---

## 🔧 РЕАЛИЗАЦИЯ:

### Backend изменения:

#### 1. Clients API (уже есть):
```typescript
GET /api/clients
- BROKER: только свои клиенты
- ADMIN: все клиенты
- DEVELOPER: 403 Forbidden
```

#### 2. Projects API:
```typescript
POST /api/projects
- DEVELOPER: создает свои проекты
- ADMIN: создает любые проекты
- BROKER: 403 Forbidden

PUT /api/projects/:id
- DEVELOPER: только свои проекты
- ADMIN: любые проекты
- BROKER: 403 Forbidden
```

#### 3. Apartments API:
```typescript
POST /api/apartments
- DEVELOPER: только в свои проекты
- ADMIN: в любые проекты
- BROKER: 403 Forbidden

PUT /api/apartments/:id
- DEVELOPER: только в своих проектах
- ADMIN: в любых проектах
- BROKER: 403 Forbidden
```

#### 4. Bookings API:
```typescript
GET /api/bookings
- BROKER: только свои брони
- DEVELOPER: брони на свои квартиры (с данными брокера)
- ADMIN: все брони

POST /api/bookings
- BROKER: может создавать
- DEVELOPER: 403 Forbidden (не работает с клиентами)
- ADMIN: может создавать

POST /api/bookings/:id/complete-deal
- BROKER: только свои брони
- DEVELOPER: брони на свои квартиры
- ADMIN: любые брони
```

#### 5. Users API (новый):
```typescript
GET /api/users
- ADMIN: все пользователи
- Others: 403 Forbidden

POST /api/users
- ADMIN: создает брокеров и девелоперов
- Others: 403 Forbidden
```
