### 4. ✅ RBAC - Разделение ролей (ОСНОВНОЕ!)

#### Backend:

**Создан:** `backend/src/routes/users.admin.routes.ts`

**API endpoints (ADMIN only):**
```
GET    /api/admin/users          - список пользователей
POST   /api/admin/users          - создать пользователя
PUT    /api/admin/users/:id      - обновить пользователя
DELETE /api/admin/users/:id      - удалить пользователя
POST   /api/admin/users/:id/reset-password - сбросить пароль
```

**Функции:**
- Создание брокеров и девелоперов
- Валидация email и роли
- Хеширование паролей (bcryptjs)
- Проверка прав доступа (только ADMIN)
- Защита от удаления себя

#### Frontend:

**Созданы страницы:**
1. `/dashboard/users` - управление пользователями (список)
2. `/dashboard/users/new` - создание пользователя

**Функции:**
- Список всех пользователей с ролями
- Статистика по ролям (брокеры, девелоперы, админы)
- Создание нового пользователя
- Удаление пользователя
- Красивые badges для ролей

## 🎯 ЛОГИКА РОЛЕЙ (ТЗ):

### BROKER (Брокер):
✅ **Может:**
- Создавать/редактировать СВОИХ клиентов
- Просматривать все проекты
- Просматривать все квартиры
- Создавать брони
- Управлять СВОИМИ бронями
- Оформлять продажу СВОИХ броней
- Использовать калькулятор

❌ **Не может:**
- Создавать/редактировать проекты
- Создавать/редактировать квартиры
- Видеть клиентов других брокеров
- Создавать пользователей

---

### DEVELOPER (Девелопер):
✅ **Может:**
- Создавать/редактировать СВОИ проекты
- Создавать/редактировать квартиры в СВОИХ проектах
- Видеть брони на СВОИ квартиры
- Видеть какой брокер забронировал квартиру
- Оформлять продажу на свои квартиры

❌ **Не может:**
- Видеть/создавать клиентов (нет доступа к CRM)
- Создавать брони
- Редактировать проекты других девелоперов
- Создавать пользователей

---

### ADMIN (Администратор):
✅ **Может:**
- **ВСЁ!**
- Создавать брокеров и девелоперов
- Видеть всех клиентов
- Видеть все проекты и квартиры
- Видеть все брони
- Редактировать любые данные
- Полная аналитика
- Управление пользователями
